### Project Vision Document (PVD)

# 1. Project Overview

Reepo is a web application that transforms GitHub repository discovery into an engaging, fast, and rewarding experience.

Instead of manually browsing GitHub, users discover open-source projects through a randomized discovery system combined with lightweight gamification.

Reepo is not a GitHub client.

Reepo is not a code hosting platform.

Reepo is a discovery platform.

---

# 2. Vision Statement

To become the most enjoyable way to discover open-source software.

---

# 3. Mission

Help developers discover projects they would never have found through traditional search.

Reduce discovery fatigue.

Encourage curiosity.

Promote open source.

---

# 4. North Star

> Make discovering open-source projects fast, enjoyable and addictive—in a positive way.

Every feature added to Reepo must support this goal.

---

# 5. Problem Statement

GitHub contains millions of repositories.

Most developers repeatedly discover the same popular projects.

Interesting repositories remain invisible because:

- Search requires knowing what to search.
- Trending pages favor already popular projects.
- Discovery is slow.
- Exploration feels like work.

Reepo removes friction by making discovery effortless.

# 15. Anti Features

Reepo will never become:

❌ GitHub clone

❌ Social media platform

❌ IDE

❌ Git client

❌ Documentation platform

❌ Project management software

❌ Repository hosting service

If a feature moves Reepo toward any of these products, it should be rejected.

---

# 16. Long-Term Vision

Reepo becomes the default destination for discovering open-source projects.

Developers visit Reepo not because they know what they are looking for...

...but because they want to be surprised.

---

# 17. Guiding Question

"Does this make discovering repositories more enjoyable?"

If the answer is no,
